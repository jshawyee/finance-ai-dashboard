# 系统架构

```text
GitHub Actions（北京时间 09:47 / 10:07 / 10:27 / 10:47 / 11:07）
        │
        ├── Yahoo Finance 公共 Chart JSON ── 收盘过滤 ── 缓存降级
        ├── GDELT 新闻索引 ──────────────── 去重与来源标记
        │
        └── 固定规则分析（无 LLM、无 Token）
                         │
                  dashboard.json
                    ┌────┴────┐
             Vue 静态站点    飞书签名机器人
                    │
               GitHub Pages
```

## 可靠性边界

- 每次只采用各市场最近一个已经结束的完整交易日。北京时间 10:00 的日本、韩国盘中数据会被排除。
- 单个代码采集失败时，保留该代码上次成功结果并标记为 `stale`；绝不以 0 或虚构值覆盖。
- 首次运行若指数或股票完全不可用，任务直接失败并保留线上旧版本。
- 免费公共源没有商业 SLA。五次错峰触发提高成功率，但 GitHub 不保证准点到秒；当天首次成功后，后续补偿任务不会重复推送。
- 新闻和行情只做相关性整理。规则分析不会将同日新闻断言为涨跌的确定原因。

## 数据状态

| 状态 | 含义 |
|---|---|
| `fresh` | 所有观察项成功取得最新完整收盘 |
| `partial` | 至少一个观察项使用缓存或暂时缺失 |
| `stale` | 单个行情来自上次成功缓存 |
| `demo` | 仅前端内置演示，在首次真实任务前使用 |
